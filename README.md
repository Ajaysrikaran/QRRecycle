# QRRecycle

## Beschreibung
QRRecycle ist eine einfache Webanwendung für Elektronikgeräte.
Durch das Scannen eines QR-Codes können Nutzer Informationen zu Materialien, Bauteilen, Reparatur und Recycling eines Geräts ansehen.

## Problem
Bei vielen Elektronikgeräten fehlen klare Informationen zu Materialien und Bauteilen.
Dadurch werden Reparatur und Recycling erschwert.
Viele Geräte werden zu früh weggeworfen und es entsteht mehr Elektroschrott.

## Lösung
QRRecycle zeigt über einen QR-Code die wichtigsten Informationen zu einem Gerät.
So können Nutzer, Reparaturdienste und Recyclingfirmen schneller verstehen, wie ein Gerät repariert oder richtig recycelt werden kann.

## Ziel
Das Ziel ist, Geräte länger zu nutzen, Reparatur zu erleichtern und Elektroschrott zu reduzieren.

## Was ist am Projekt neu
Das Projekt verbindet QR-Code, Materialinformationen und Recycling in einer einfachen digitalen Lösung.
Die Informationen sind schnell zugänglich und direkt mit einem Gerät verknüpft.

## Ist das Produkt intuitiv
Ja.
Der Nutzer muss nur einen QR-Code scannen und bekommt direkt die wichtigsten Informationen angezeigt.

## Wirkt das Produkt klar oder verwirrend
Das Produkt soll klar und einfach wirken.
Die Oberfläche soll übersichtlich sein und nur die wichtigsten Informationen zeigen.

## Braucht man eine Anleitung
Nein.
QR-Codes scannen kennen viele Nutzer bereits.
Darum soll die Anwendung ohne grosse Erklärung verständlich sein.

## Was könnte man weglassen
Zusatzfunktionen, die für den ersten Prototyp nicht nötig sind.
Zum Beispiel Login, Bewertungssystem, Kommentare oder ein kompliziertes Design.

## MVP Funktionen
QR-Code scannen
Materialinformationen anzeigen
Bauteile anzeigen
Hinweise zur Reparatur anzeigen
Hinweise zum Recycling anzeigen

## Zielgruppe
Nutzer von Elektronikgeräten
Reparaturdienste
Recyclingfirmen
Hersteller

## Nutzen
Bessere Transparenz über Geräte
Einfachere Reparatur
Einfacheres Recycling
Längere Nutzung von Geräten
Weniger Elektroschrott

## Anforderungen
Die Anwendung soll einfach verständlich sein.
Die Informationen sollen schnell sichtbar sein.
Die Nutzung soll ohne Anleitung möglich sein.
Der Fokus liegt auf den wichtigsten Daten zum Gerät.
